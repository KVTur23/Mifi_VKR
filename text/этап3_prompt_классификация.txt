1.2    Prompt-based классификация на основе генеративных LLM

    Третий этап системы представляет собой альтернативный подход к классификации входящей корреспонденции, принципиально отличающийся от классических методов машинного обучения. Вместо обучения отдельного классификатора на размеченных данных, prompt-based подход использует инструкционно-настроенные генеративные языковые модели (instruction-tuned LLM) в режиме вывода. Модели предъявляется текст письма вместе с описаниями всех 36 подразделений и примерами (в режиме few-shot), после чего модель генерирует название подходящего подразделения на основании семантического сопоставления.

    Выбор данного подхода обусловлен двумя факторами. Во-первых, при малом количестве размеченных примеров (классы группы C, менее 15 писем) традиционные классификаторы демонстрируют нестабильное качество. Во-вторых, генеративные LLM обладают обширными знаниями о предметных областях нефтегазовой промышленности, что позволяет компенсировать дефицит обучающих примеров.

    Реализованный пайплайн состоит из пяти шагов: генерация описаний классов с помощью LLM; подготовка few-shot примеров с учётом группы дефицита данных; построение промптов трёх типов (zero-shot, one-shot, few-shot); генерация и постобработка ответов модели; оценка результатов с разбивкой по группам классов.

    Выбор моделей обусловлен их специализацией на русскоязычных задачах. Семейство Saiga (saiga_llama3_8b и saiga_mistral_7b) представляет собой русскоязычные fine-tune на базе LLaMA3 и Mistral с instruction-following настройкой. Vikhr-Nemo-12B — более крупная модель от команды Vikhr Models. Семейство Qwen2.5 (14B и 32B) — многоязычные модели от Alibaba, демонстрирующие высокое качество на русском языке. Модель Qwen-32B загружается в формате AWQ-квантизации (Activation-aware Weight Quantization), что позволяет уместить её в 20 ГБ видеопамяти против 65 ГБ при полной точности. Низкая температура генерации (temperature=0.1) намеренно снижает вариативность ответов, поскольку задача классификации не требует творческого разнообразия.

При первом запуске они генерируются моделью Qwen-32B-AWQ: для каждого из 36 подразделений модели предъявляется название класса и до пяти реальных писем из обучающей выборки, после чего генерируется краткое семантическое описание. В последующих запусках описания загружаются из кэша через функцию load_class_descriptions() модуля prompt_classifier.py:

1.2.2   Модуль подготовки few-shot примеров (few_shot_examples.py)

    Для режимов one-shot и few-shot необходимо сформировать набор демонстрационных примеров для каждого класса. Модуль few_shot_examples.py реализует стратегию отбора примеров, основанную на прямом сравнении количества доступных оригинальных текстов с требуемым K.

    Функция classify_groups() вычисляет принадлежность каждого класса к группе A/B/C и сохраняет её в JSON исключительно для подсчёта per-group метрик на этапе оценки — на логику отбора примеров группы не влияют:

def classify_groups(df_original: pd.DataFrame) -> dict[str, str]:
    counts = df_original[LABEL_COL].value_counts()
    groups = {}
    for cls, n in counts.items():
        if n >= GROUP_A_MIN:    # >= 50
            groups[cls] = "A"
        elif n >= GROUP_B_MIN:  # 15–49
            groups[cls] = "B"
        else:                   # < 15
            groups[cls] = "C"
    return groups

    Для корпуса используемого датасета распределение по группам составило: группа A (богатые данные, 50+ примеров) — 9 классов; группа B (средние данные, 15–49) — 15 классов; группа C (дефицитные классы, менее 15) — 12 классов.

    Функция select_examples() отбирает K примеров для каждого класса. Решение о подмешивании аугментированных данных принимается индивидуально для каждого класса: если оригинальных примеров хватает для заполнения K слотов — аугментированные не используются; если нет — добираются аугментированные тексты, не пересекающиеся с оригинальными. Такой подход точнее предыдущего порогового (группы A/B/C): анализ корпуса показывает, что из 12 классов группы C лишь 6 реально не могут обеспечить K=5 оригинальными примерами (классы с ≤4 письмами), тогда как остальные 6 классов группы C имеют 5–13 оригинальных примеров и не требуют дополнения:

def select_examples(
    df_original: pd.DataFrame,
    df_augmented: pd.DataFrame,
    k: int,
    seed: int = RANDOM_SEED,
) -> dict[str, list[str]]:
    rng = np.random.RandomState(seed)
    examples = {}

    for cls in sorted(df_original[LABEL_COL].unique()):
        orig_texts = df_original[df_original[LABEL_COL] == cls][TEXT_COL].tolist()

        if len(orig_texts) >= k:
            pool = orig_texts           # оригинальных хватает — аугментацию не трогаем
        else:
            # Оригинальных меньше K — добираем аугментированными
            aug_texts = df_augmented[
                (df_augmented[LABEL_COL] == cls)
                & (~df_augmented[TEXT_COL].isin(orig_texts))
            ][TEXT_COL].tolist()
            pool = orig_texts + aug_texts

        pool_shuffled = list(pool)
        rng.shuffle(pool_shuffled)
        examples[cls] = pool_shuffled[:k]

    return examples

    Функция-оркестратор prepare_few_shot_examples() запускает отбор для K∈{1, 3, 5} и сохраняет результат в Data/few_shot_examples.json. Структура файла содержит два ключа: groups (словарь {класс: группа} для метрик) и examples (словарь {K: {класс: [тексты]}}):

def prepare_few_shot_examples(
    k_values: list[int] = None,
    save_path: str | Path | None = None,
) -> dict:
    df_original = load_dataset(stage=0)   # оригинальный train
    df_augmented = load_dataset(stage=3)  # аугментированный train

    groups = classify_groups(df_original)
    result = {"groups": groups, "examples": {}}

    for k in k_values:
        examples = select_examples(df_original, df_augmented, k)
        result["examples"][str(k)] = examples

    with open(save_path, "w", encoding="utf-8") as f:
        json.dump(result, f, ensure_ascii=False, indent=2)

    return result


1.2.3   Шаблоны промптов

    Каждому из трёх режимов классификации соответствует отдельный текстовый файл в директории prompts/classification_prompts/. Шаблоны содержат плейсхолдеры Python-формата и заполняются функцией build_prompt() модуля prompt_classifier.py.

    Шаблон zero_shot.txt не содержит демонстрационных примеров и опирается исключительно на описания подразделений:

Ты — система маршрутизации входящей корреспонденции нефтегазовой компании.

Твоя задача — определить РОВНО ОДНО подразделение, наиболее подходящее по смыслу письма.

Список подразделений:
{class_list}

Описания подразделений (используй их как основной критерий выбора):
{class_descriptions}

Правила:
* Выбери РОВНО ОДИН вариант из списка подразделений
* Не придумывай новые варианты
* Если несколько вариантов похожи, выбери наиболее близкий по смыслу
* Всегда делай выбор, даже если уверенность низкая

Ответ должен содержать ТОЛЬКО название подразделения из списка, без пояснений.

Письмо:
{text}

Подразделение:

    Шаблон one_shot.txt добавляет блок с одним демонстрационным примером перед классифицируемым письмом:

...
Ответ должен содержать ТОЛЬКО название подразделения из списка, без пояснений.

Пример:

Письмо: {example_text}
Подразделение: {example_label}

Письмо:
{text}

Подразделение:

    Шаблон few_shot.txt заменяет единственный пример блоком из K примеров для каждого класса:

Ответ должен содержать ТОЛЬКО название подразделения из списка, без пояснений.

Примеры классификации:

{examples_block}

Письмо:
{text}

Подразделение:

    Ключевым решением в дизайне промптов является явное ограничение формата вывода ("Ответ должен содержать ТОЛЬКО название подразделения из списка, без пояснений") и инструкция всегда делать выбор. Без этого ограничения LLM склонна генерировать развёрнутое рассуждение, что существенно затрудняет парсинг ответа. Низкая температура (0.1) дополнительно снижает вариативность форматирования ответа.


1.2.4   Модуль prompt_classifier.py: построение промптов

    Модуль prompt_classifier.py является центральным в пайплайне prompt-based классификации. Функция format_class_list() формирует нумерованный список из 36 классов в алфавитном порядке; format_class_descriptions() форматирует описания, обрезая каждое до 200 символов для экономии контекста; format_examples_block() формирует блок few-shot примеров, обрезая каждый текст до 500 символов:

def format_class_list(labels: list[str]) -> str:
    return "\n".join(f"{i+1}. {label}" for i, label in enumerate(sorted(labels)))

def format_class_descriptions(descriptions: dict[str, str], max_chars: int = 200) -> str:
    lines = []
    for cls in sorted(descriptions.keys()):
        desc = descriptions[cls]
        if len(desc) > max_chars:
            desc = desc[:max_chars].rsplit(" ", 1)[0] + "..."
        lines.append(f"- {cls}: {desc}")
    return "\n".join(lines)

def format_examples_block(examples: dict[str, list[str]]) -> str:
    lines = []
    for cls in sorted(examples.keys()):
        for text in examples[cls]:
            truncated = text[:500] + "..." if len(text) > 500 else text
            lines.append(f"Письмо: {truncated}\nПодразделение: {cls}")
            lines.append("")
    return "\n".join(lines)

    Функция build_prompt() является фасадом, принимающим режим классификации и объединяющим все компоненты в итоговый промпт:

def build_prompt(
    text: str,
    labels: list[str],
    descriptions: dict[str, str],
    mode: str = "zero_shot",
    examples: Optional[dict[str, list[str]]] = None,
) -> str:
    template = load_prompt_template(mode)
    class_list = format_class_list(labels)
    class_desc = format_class_descriptions(descriptions)

    if mode == "zero_shot":
        return template.format(
            class_list=class_list, class_descriptions=class_desc, text=text,
        )
    elif mode == "one_shot":
        first_cls = sorted(examples.keys())[0]
        ex_text = examples[first_cls][0] if examples[first_cls] else ""
        ex_text_trunc = ex_text[:500] + "..." if len(ex_text) > 500 else ex_text
        return template.format(
            class_list=class_list, class_descriptions=class_desc,
            example_text=ex_text_trunc, example_label=first_cls, text=text,
        )
    else:  # few_shot
        examples_block = format_examples_block(examples)
        return template.format(
            class_list=class_list, class_descriptions=class_desc,
            examples_block=examples_block, text=text,
        )


1.2.5   Загрузка моделей и генерация ответов

    Функция load_model() загружает модель через библиотеку transformers с автоматическим распределением по устройствам (device_map="auto"). Для AWQ-моделей (Qwen-32B) параметр torch_dtype не передаётся — библиотека autoawq определяет тип данных автоматически:

def load_model(model_cfg: dict) -> tuple:
    from transformers import AutoModelForCausalLM, AutoTokenizer

    tokenizer = AutoTokenizer.from_pretrained(
        model_cfg["model_name"], trust_remote_code=True,
    )
    if tokenizer.pad_token is None:
        tokenizer.pad_token = tokenizer.eos_token

    dtype_map = {"float16": torch.float16, "bfloat16": torch.bfloat16,
                 "float32": torch.float32}
    dtype = dtype_map.get(model_cfg.get("dtype", "float16"), torch.float16)

    load_kwargs = {"trust_remote_code": True, "device_map": "auto"}
    if model_cfg.get("quantization") != "awq":
        load_kwargs["torch_dtype"] = dtype

    model = AutoModelForCausalLM.from_pretrained(model_cfg["model_name"], **load_kwargs)
    model.eval()
    return model, tokenizer

    Перед подачей промпта в модель функция apply_chat_template() оборачивает его в conversation-формат, специфичный для каждой instruction-tuned модели. Модели Saiga, Vikhr и Qwen ожидают входные данные в формате с ролями ("role": "user"), и передача сырого текста без этого форматирования приводит к деградации качества:

def apply_chat_template(tokenizer, prompt: str) -> str:
    messages = [{"role": "user", "content": prompt}]
    try:
        formatted = tokenizer.apply_chat_template(
            messages, tokenize=False, add_generation_prompt=True,
        )
        return formatted
    except Exception:
        return prompt  # Fallback для моделей без chat template

    Функция generate_response() выполняет проверку длины контекста перед генерацией. Если токенизированный промпт занимает более max_context − 100 токенов, функция возвращает None и пример помечается как skipped. Декодируются только новые токены (output[input_len:]), что исключает повторение входного промпта в ответе:

def generate_response(
    model, tokenizer, prompt: str, gen_params: dict, max_context: int = 8192,
) -> Optional[str]:
    formatted_prompt = apply_chat_template(tokenizer, prompt)
    inputs = tokenizer(formatted_prompt, return_tensors="pt", truncation=False)
    input_len = inputs["input_ids"].shape[1]

    if input_len >= max_context - 100:
        return None  # Промпт не влезает в контекст

    inputs = {k: v.to(model.device) for k, v in inputs.items()}
    with torch.no_grad():
        outputs = model.generate(
            **inputs,
            max_new_tokens=gen_params.get("max_new_tokens", 50),
            temperature=gen_params.get("temperature", 0.1),
            do_sample=gen_params.get("do_sample", True),
            top_p=gen_params.get("top_p", 0.9),
            repetition_penalty=gen_params.get("repetition_penalty", 1.1),
            pad_token_id=tokenizer.pad_token_id,
        )

    new_tokens = outputs[0][input_len:]
    return tokenizer.decode(new_tokens, skip_special_tokens=True).strip()

    После завершения всех экспериментов с одной моделью вызывается функция unload_model(), освобождающая GPU-память перед загрузкой следующей модели:

def unload_model(model, tokenizer):
    del model
    del tokenizer
    gc.collect()
    if torch.cuda.is_available():
        torch.cuda.empty_cache()


1.2.6   Извлечение предсказания из ответа модели

    Несмотря на ограничение формата в промпте, модели нередко добавляют пояснения, нумерацию или используют нестандартные кавычки. Для надёжного извлечения названия подразделения реализована многоуровневая стратегия в функции extract_prediction().

    Вспомогательная функция _normalize() приводит текст к единому регистру и убирает пунктуацию, а _clean_response() извлекает кандидатов из первых трёх строк ответа, удаляя типичные для LLM-ответов префиксы:

def _normalize(text: str) -> str:
    text = text.lower()
    text = re.sub(r'[^а-яёa-z0-9\s]', ' ', text)
    text = re.sub(r'\s+', ' ', text).strip()
    return text

def _clean_response(response: str) -> list[str]:
    candidates = []
    lines = [l.strip() for l in response.strip().split('\n') if l.strip()]

    for line in lines[:3]:  # Проверяем первые 3 строки
        cleaned = line
        prefixes = [
            r'^подразделение\s*:\s*', r'^ответ\s*:\s*', r'^результат\s*:\s*',
            r'^категория\s*:\s*',     r'^отдел\s*:\s*',  r'^\d+[\.\)]\s*',
            r'^[-*•]\s*',
        ]
        for prefix in prefixes:
            cleaned = re.sub(prefix, '', cleaned, flags=re.IGNORECASE)
        cleaned = cleaned.strip().strip('"«»""\'\'').strip('.,:;!').strip()
        if cleaned:
            candidates.append(cleaned)

    return candidates

    Стратегия сопоставления кандидата с эталонным классом работает последовательно — от точных методов к нечётким:

def extract_prediction(
    response: str,
    labels: list[str],
    fuzzy_cutoff: float = 0.6,
) -> str:
    label_norm = {_normalize(l): l for l in labels}
    candidates = _clean_response(response)
    if not candidates:
        return "unknown"

    for candidate in candidates:
        cand_norm = _normalize(candidate)

        # Уровень 1: точное нормализованное совпадение
        if cand_norm in label_norm:
            return label_norm[cand_norm]

        # Уровень 2: нормализованный label содержится в кандидате
        partial_matches = [label for lnorm, label in label_norm.items()
                           if lnorm in cand_norm]
        if partial_matches:
            return max(partial_matches, key=len)

        # Уровень 3: кандидат содержится в label (короткий ответ)
        if len(cand_norm) > 5:
            for lnorm, label in label_norm.items():
                if cand_norm in lnorm:
                    return label

    # Уровень 4: нечёткое совпадение через difflib (cutoff=0.6)
    best_candidate = _normalize(candidates[0])
    matches = difflib.get_close_matches(
        best_candidate, label_norm.keys(), n=1, cutoff=fuzzy_cutoff,
    )
    if matches:
        return label_norm[matches[0]]

    # Уровень 5: поиск любого label в полном тексте ответа
    response_norm = _normalize(response)
    partial_in_full = [label for lnorm, label in label_norm.items()
                       if lnorm in response_norm]
    if partial_in_full:
        return max(partial_in_full, key=len)

    return "unknown"

    Параметр fuzzy_cutoff=0.6 выбран как компромисс между полнотой (не пропустить корректный ответ, записанный с опечаткой) и точностью (не принять ошибочное совпадение). При unknown rate > 0 это означает, что ни один из уровней сопоставления не сработал.


1.2.7   Основной пайплайн и матрица экспериментов

    Функция classify_dataset() итерирует по тестовой выборке (343 примера), строя промпт и генерируя ответ для каждого письма, и возвращает DataFrame с колонками text, true_label, predicted_label, raw_response, skipped:

def classify_dataset(
    df_test: pd.DataFrame,
    model, tokenizer,
    labels: list[str],
    descriptions: dict[str, str],
    mode: str,
    gen_params: dict,
    max_context: int,
    examples: Optional[dict[str, list[str]]] = None,
    fuzzy_cutoff: float = 0.7,
) -> pd.DataFrame:
    results = []

    for _, row in tqdm(df_test.iterrows(), total=len(df_test), desc=f"Classify ({mode})"):
        text = row[TEXT_COL]
        true_label = row[LABEL_COL]

        prompt = build_prompt(text, labels, descriptions, mode, examples)
        response = generate_response(model, tokenizer, prompt, gen_params, max_context)

        if response is None:
            results.append({"text": text, "true_label": true_label,
                            "predicted_label": "unknown", "raw_response": None,
                            "skipped": True})
        else:
            pred = extract_prediction(response, labels, fuzzy_cutoff)
            results.append({"text": text, "true_label": true_label,
                            "predicted_label": pred, "raw_response": response,
                            "skipped": False})

    return pd.DataFrame(results)

    В ноутбуке zer0_one_few_shot.ipynb эксперименты организованы в виде матрицы «модель × K». Модель Saiga-LLaMA3-8B тестируется только при K=0 и K=1 ввиду ограниченного размера контекста (8192 токена) — промпт few-shot с 3–5 примерами на каждый из 36 классов превышает этот предел. Остальные четыре модели с контекстом 32768 токенов тестируются при всех K∈{0, 1, 3, 5}:

experiment_matrix = {
    "saiga_8b":          [0, 1],
    "saiga_mistral_7b":  [0, 1, 3, 5],
    "vikhr_12b":         [0, 1, 3, 5],
    "qwen_14b":          [0, 1, 3, 5],
    "qwen_32b":          [0, 1, 3, 5],
}

    Каждый эксперимент идентифицируется парой (model_key, k_shots) и сохраняется в prompt_results.csv немедленно после завершения. При перезапуске ноутбука уже выполненные эксперименты пропускаются:

# Подхватываем уже посчитанные результаты
if results_path.exists():
    df_existing = pd.read_csv(results_path)
    all_results = df_existing.to_dict("records")
    done = {(r["model"], int(r["k_shots"])) for r in all_results}

# Основной цикл
for model_key, k_values in experiment_matrix.items():
    model_cfg = prompt_models[model_key]
    k_todo = [k for k in k_values if (model_key, k) not in done]

    if not k_todo:
        print(f"[SKIP] {model_cfg['model_name']} — все K уже посчитаны")
        continue

    try:
        model, tokenizer = load_model(model_cfg)
    except Exception as e:
        print(f"[ERROR] Не удалось загрузить {model_cfg['model_name']}: {e}")
        continue

    for k in k_todo:
        result = run_experiment(model_key, k, model, tokenizer, model_cfg)
        all_results.append(result)
        pd.DataFrame(all_results).to_csv(results_path, index=False)

    unload_model(model, tokenizer)

    Такая архитектура позволяет прерывать и возобновлять вычисления без потери уже полученных результатов, что критично при работе на облачных GPU с ограниченным временем сессии.


1.2.8   Оценка результатов (evaluate.py)

    Для оценки prompt-based классификации в модуль evaluate.py добавлена специализированная функция evaluate_prompt_classification(). В отличие от стандартных sklearn-метрик, она обрабатывает специфику задачи: отфильтровывает skipped-примеры (промпт не уложился в контекст), учитывает unknown rate (доля ответов, не сопоставленных ни с одним классом) и вычисляет per-group метрику Macro F1 отдельно для групп A, B и C:

def evaluate_prompt_classification(
    df_results: pd.DataFrame,
    groups: dict[str, str] | None = None,
) -> dict:
    n_total = len(df_results)
    n_skipped = int(df_results["skipped"].sum())

    df = df_results[~df_results["skipped"]].copy()
    y_true = df["true_label"].tolist()
    y_pred = df["predicted_label"].tolist()
    all_labels = sorted(set(y_true))

    bal_acc = balanced_accuracy_score(y_true, y_pred)
    f1_mac  = f1_score(y_true, y_pred, average="macro",
                       zero_division=0, labels=all_labels)

    n_unknown = sum(1 for p in y_pred if p == "unknown")
    unknown_rate = n_unknown / len(y_pred) if y_pred else 0.0

    result = {
        "balanced_accuracy": bal_acc, "macro_f1": f1_mac,
        "unknown_rate": unknown_rate,
        "n_test": n_total, "n_skipped": n_skipped,
        "classification_report": classification_report(y_true, y_pred,
            labels=all_labels, zero_division=0),
    }

    # Per-group метрики
    if groups:
        for group_name in ("A", "B", "C"):
            group_classes = [c for c, g in groups.items() if g == group_name]
            mask = df["true_label"].isin(group_classes)
            if mask.sum() > 0:
                df_g = df[mask]
                f1_g = f1_score(df_g["true_label"], df_g["predicted_label"],
                    average="macro", zero_division=0,
                    labels=sorted(set(df_g["true_label"])))
            else:
                f1_g = 0.0
            result[f"f1_group_{group_name}"] = f1_g

    return result

    Per-group метрики позволяют проверить центральную гипотезу исследования: prompt-based подход должен давать относительно более высокое качество именно на классах группы C (дефицитные данные), где классические классификаторы страдают от нехватки обучающих примеров, тогда как LLM опирается на семантические описания подразделений, не зависящие от размера обучающей выборки.
