## Конфигурация LLM-моделей

Каждая модель описывается отдельным JSON-файлом в `configs/`. Пример `configs/model_qwen.json`:

```json
{
  "model_name": "Qwen/Qwen2.5-7B-Instruct",
  "generation_params": {
    "temperature": 0.7,
    "top_p": 0.9,
    "max_new_tokens": 512,
    "repetition_penalty": 1.1,
    "do_sample": true
  },
  "prompt_template": "llm_generate.txt"
}
```

**Правила:**
- Поле `model_name` — любая модель с HuggingFace (transformers-совместимая).
- `generation_params` — передаются напрямую в `model.generate()`.
- `prompt_template` — имя файла из папки `prompts/`.
- Функция загрузки модели в `llm_utils.py` должна быть **универсальной**: принимает путь до JSON-конфига, загружает модель и токенизатор через `AutoModelForCausalLM` / `AutoTokenizer`.

## Пайплайн аугментации

### Общие принципы

1. **Чекпоинтинг:** Перед каждым этапом проверяй наличие промежуточного файла (`data_after_stageN.csv`). Если файл есть — загружай его, анализируй распределение классов и пропускай уже завершённые шаги.
2. **Сохранение:** После каждого этапа сохраняй полный обновлённый датасет в промежуточный файл.
3. **Повтор при отсеве:** Если валидация отсеяла сгенерированные примеры и целевое количество не достигнуто — повторить генерацию для недостающих примеров.
4. **Не трогать классы с ≥50 примерами** — они не участвуют в аугментации.

### Разбиение классов по группам

| Группа | Кол-во примеров в классе | Целевое кол-во | Метод аугментации |
|---|---|---|---|
| A | < 15 | 15 | Этап 1: LLM-генерация |
| B | 15–34 | 35 | Этап 2: парафраз через LLM |
| C | 35–49 | 50 | Этап 3: обратный перевод |
| D | ≥ 50 | без изменений | — |

**Важно:** после каждого этапа группы пересчитываются. Например, класс из группы A после этапа 1 попадает в группу B и участвует в этапе 2.

### Этап 1: LLM-генерация (< 15 → 15)

- **Модуль:** `src/augmentation/stage1_llm_generate.py`
- **Метод:** Генерация новых текстов через LLM (HuggingFace-модель из JSON-конфига).
- **Вход:** классы с < 15 примерами.
- **Выход:** `Data/data_after_stage1.csv` — все классы имеют ≥ 15 примеров.
- **LLM-функция:** из `llm_utils.py`, принимает путь до JSON-конфига как аргумент.

### Этап 2: Парафраз через LLM (< 35 → 35)

- **Модуль:** `src/augmentation/stage2_paraphrase.py`
- **Метод:** Перефразирование существующих текстов через LLM — модель получает оригинальный текст и генерирует его переформулировку с сохранением смысла, но с другой структурой предложений, синонимами, порядком слов.
- **Вход:** классы с 15–34 примерами (включая дополненные на этапе 1).
- **Выход:** `Data/data_after_stage2.csv` — все классы имеют ≥ 35 примеров.
- **LLM-функция:** из `llm_utils.py`, тот же конфиг что и на этапе 1 (или отдельный). Промпт — `prompts/paraphrase.txt`.

### Этап 3: Обратный перевод (< 50 → 50)

- **Модуль:** `src/augmentation/stage3_back_translation.py`
- **Метод:** Обратный перевод через Helsinki-NLP модели (HuggingFace, офлайн): RU → EN → RU.
- **Модели перевода:**
  - `Helsinki-NLP/opus-mt-ru-en` (RU → EN)
  - `Helsinki-NLP/opus-mt-en-ru` (EN → RU)
- **Вход:** классы с 35–49 примерами.
- **Выход:** `Data/data_after_stage3.csv` — все классы имеют ≥ 50 примеров.

## Валидация после каждого этапа

**Модуль:** `src/augmentation/validation.py`

После каждого этапа аугментации запускается валидация сгенерированных текстов. Все проверки применяются **только к новым (сгенерированным) текстам** относительно оригинальных.

### Проверки:

1. **Точные дубликаты** — удалить полные совпадения с существующими текстами и между собой.
2. **Косинусное сходство** — вычислить эмбеддинги через BERT-модель, отсеять тексты с косинусным сходством > порога (например, 0.95) к любому существующему тексту того же класса.
3. **Минимальная длина** — отсеять слишком короткие тексты (< N символов, настраиваемый порог).
4. **Проверка языка** — убедиться, что текст на русском (актуально после обратного перевода).
5. **Проверка на вырожденность** — отсеять тексты, состоящие из повторов, бессмысленных последовательностей.

### BERT-модель для эмбеддингов

Рекомендуемая модель: **`ai-forever/sbert_large_nlu_ru`**.

Обоснование: это специализированная sentence-BERT модель, обученная на русском языке для задач семантического сходства. Даёт качественные эмбеддинги предложений для русского текста и хорошо подходит для косинусной проверки. Альтернатива — `intfloat/multilingual-e5-large` (мультиязычная, чуть тяжелее, но универсальнее).

### Логика повтора

```
повторить:
    сгенерировать недостающие примеры
    провести валидацию
    если после валидации целевое кол-во достигнуто → выход
    если нет → повторить генерацию для недостающих
    если превышен лимит попыток (например, 5) → предупредить и продолжить
```

## Классификация (baseline-модели)

Цель — оценить качество аугментации через простые модели. Каждая модель — отдельный скрипт, все взаимозаменяемы.

### Общий пайплайн классификации:

1. Загрузить финальный датасет (`data_after_stage3.csv`).
2. Получить эмбеддинги текстов через `ai-forever/sbert_large_nlu_ru` (модуль `embeddings.py`).
3. Обучить модель с кросс-валидацией (StratifiedKFold, k=5).
4. Вывести метрики: accuracy, macro F1, weighted F1, classification report.

### Модели:

| Модуль | Модель | Примечания |
|---|---|---|
| `run_svm.py` | LinearSVC | С подбором гиперпараметров (C) |
| `run_logreg.py` | LogisticRegression | multi_class='multinomial' |
| `run_naive_bayes.py` | MultinomialNB / GaussianNB | GaussianNB для dense-эмбеддингов |

Каждый скрипт запускается автономно и выводит результаты в консоль/файл.

## Правила для Claude Code

### Модульность и архитектура

1. **Модульность:** каждый этап — отдельный скрипт, модули не зависят друг от друга (кроме общих утилит).
2. **Чекпоинтинг обязателен:** перед каждым этапом проверяй промежуточный файл. Никогда не перезаписывай данные без проверки.
3. **LLM-обёртка универсальна:** `llm_utils.py` должна работать с любой HuggingFace-моделью — не хардкодить имена моделей.
4. **Валидация после каждого этапа:** никогда не пропускай валидацию, даже если кажется, что данные чистые.
5. **Не трогай классы с ≥50 примерами.**
6. **Воспроизводимость:** фиксируй random seed (42) везде, где есть стохастика.
7. **Отказоустойчивость:** оборачивай генерацию в try/except, логируй ошибки, продолжай с другими классами.

### Стиль комментариев и документирование кода

8. **Язык:** весь код, комментарии, docstring'и, логи — на русском языке.
9. **Человечные комментарии:** пиши комментарии так, как писал бы живой разработчик для коллеги — понятно, по делу, без канцелярита и без шаблонных фраз вроде «данная функция осуществляет». Хороший комментарий объясняет **зачем**, а не **что**. Примеры:

```python
# Плохо:
# Загружаем данные из CSV-файла
df = pd.read_csv(path)

# Хорошо:
# Пробуем подхватить чекпоинт — если этап уже проходили, не будем гонять генерацию заново
df = pd.read_csv(path)

# Плохо:
# Вычисляем косинусное сходство
sim = cosine_similarity(vec_a, vec_b)

# Хорошо:
# Отсеиваем сгенерированные тексты, которые слишком похожи на уже имеющиеся —
# иначе модель просто запомнит дубликаты вместо того чтобы учиться обобщать
sim = cosine_similarity(vec_a, vec_b)
```

10. **Каждый файл начинается с шапки** — краткое описание модуля: что делает, зачем нужен, как вписывается в пайплайн. Формат:

```python
"""
stage1_llm_generate.py — Этап 1: генерация текстов через LLM

Берём классы, в которых меньше 15 примеров, и догоняем их до 15
с помощью языковой модели (конкретная модель задаётся через JSON-конфиг).
После генерации запускается валидация — дубликаты и мусор отсеиваются.
Если после отсева не хватает примеров, генерация повторяется.

Вход:  Data/data_after_eda.csv  (или data_after_stage1.csv, если чекпоинт есть)
Выход: Data/data_after_stage1.csv
"""
```

11. **Каждая функция — с docstring:** объясни что функция делает, какие аргументы принимает и что возвращает. Не формально, а по-человечески:

```python
def generate_samples(class_name: str, existing_texts: list[str], n_needed: int, llm) -> list[str]:
    """
    Генерирует новые тексты для конкретного класса через LLM.

    Получает на вход имеющиеся примеры класса и количество, которое нужно
    догенерировать. Передаёт в LLM промпт с примерами и просит сгенерировать
    новые тексты в похожем стиле. Возвращает список сгенерированных текстов
    (без валидации — она будет позже, отдельно).

    Аргументы:
        class_name:     название класса (label)
        existing_texts: уже имеющиеся тексты этого класса
        n_needed:       сколько новых текстов нужно сгенерировать
        llm:            загруженная LLM-модель (из llm_utils)

    Возвращает:
        Список строк — сгенерированные тексты
    """
```

12. **Подписывай всё, что неочевидно:** магические числа, пороги, выбор конкретной модели или метода — всегда объясняй почему именно так. Например:

```python
SIMILARITY_THRESHOLD = 0.95  # Порог косинусного сходства: выше — почти дубликат, отсеиваем
MIN_TEXT_LENGTH = 20         # Короче 20 символов — скорее всего мусор или обрезанное письмо
MAX_RETRIES = 5              # Сколько раз пробуем догенерировать, если валидация отсеяла слишком много
```

13. **Логирование:** каждый этап должен выводить человекочитаемые логи: сколько классов обрабатывается, сколько примеров нужно сгенерировать, сколько прошло валидацию, сколько отсеяно. Пример:

```python
print(f"[Этап 1] Класс '{class_name}': есть {len(existing)} писем, нужно ещё {n_needed}")
print(f"[Этап 1] Сгенерировано: {n_generated}, прошло валидацию: {n_valid}, отсеяно: {n_rejected}")
```

14. **Inline-комментарии:** не к каждой строке, но к каждому логическому блоку. Код должен читаться как рассказ — что происходит и почему:

```python
# --- Загрузка и проверка чекпоинта ---
# Если предыдущий запуск уже дошёл до этого этапа, подхватываем результат
if checkpoint_path.exists():
    df = pd.read_csv(checkpoint_path)
    print(f"Найден чекпоинт: {checkpoint_path}, загружено {len(df)} записей")
else:
    df = pd.read_csv(source_path)
    print(f"Чекпоинт не найден, работаем с исходными данными: {len(df)} записей")

# --- Определяем, каким классам нужна аугментация ---
# Считаем, сколько примеров в каждом классе, и отбираем те, где меньше порога
class_counts = df[TARGET_COL].value_counts()
classes_to_augment = class_counts[class_counts < target_count].index.tolist()
```
